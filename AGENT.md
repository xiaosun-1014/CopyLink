# AGENT.md

## 工作环境

- 默认使用 `conda` 环境 `gui`。
- 进入项目后，若需要运行代码或测试，请优先在该环境中执行。

## 编码协作方式

- 在开始写代码前，可以先使用 `superpower` 相关能力来帮助梳理任务、明确目标和拆分步骤。
- 在设计不够确定、需要严格追问边界条件或方案权衡时，可以使用 `grill-me` 来把需求问清楚。
- 代码实现时，默认遵循 `karpathy-guidelines`：
  - 保持实现尽量简单直接。
  - 优先解决真实问题，避免过度设计。
  - 先做可验证、可运行的最小改动，再逐步迭代。

## 额外说明

- 如果项目里还有更具体的约定，以更具体的约定为准。
- 如果某个任务不需要额外澄清，就直接推进实现，不要为了提问而拖慢进度。
